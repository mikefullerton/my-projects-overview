# Always Commit and Push

After every file change (create, edit, delete), immediately commit and push.

- Never leave uncommitted changes in the working tree.
- Never leave commits unpushed to origin.
- One logical change = one commit + push. Do not batch unrelated changes.
- This is non-negotiable. The remote must always reflect the current local state.
